## Codebase Patterns
- SDK lives in sdk/openreporting/ with modules: client.py, sections.py, prompts.py, models.py, validation.py, capabilities.py, scaffold.py
- capabilities.py defines THEMES, LAYOUTS, SECTION_TYPES, CHART_TYPES, CONTENT_FORMATS, CONTENT_TYPES, CATEGORIES as tuples
- prompts.py builds system prompts for LLMs; slideshow guidance is inline around lines 41-51
- sections.py has helper functions for each section type; slide() is at line 144
- validation.py validates section structure locally before sending to server
- scaffold.py generates a starter agent project with pyproject.toml, config, and agent.py
- Backend themes.py has 7 themes: default, dark, executive, financial, consulting, technical, editorial
- Backend renderers.py ignores background_color on slides — uses theme slide tokens instead
- SKILL.md v6.2.0 documents all 7 themes and theme-controlled slide backgrounds
- prompts.py: _FORMAT_INSTRUCTION = theme/layout guidance; _CATEGORY_INSTRUCTION = slideshow/category guidance
